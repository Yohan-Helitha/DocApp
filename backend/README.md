# Backend (Microservices Architecture)

Each service is independently deployable and should expose REST APIs.

## Service locations
- `services/auth-service`
- `services/patient-management-service`
- `services/doctor-management-service`
- `services/appointment-service`
- `services/telemedicine-service`
- `services/payment-service`
- `services/notification-service`
- `services/admin-management-service`
- `services/ai-symptom-checker-service` (optional)

Docker and Kubernetes deployment assets are under `infra/`.
